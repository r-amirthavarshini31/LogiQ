Getting Started with LogiQ

LogiQ is an intelligent log monitoring dashboard designed to explain system anomalies and execute self-healing remediation paths in seconds. By coupling parsing logic with an AI engine and an SQLite cache, LogiQ provides cost-efficient incident investigation.

Workflow Overview

To inspect your infrastructure systems, follow this workflow:

  1. Upload Logs: Drag and drop your `.log`, `.txt`, or `.csv` system log file onto the Upload page or select a preloaded scenarios model.
  2. Analyze Anomaly: Interact with the dashboard charts to examine spike occurrence intervals. Click "Explain" on any anomaly row to query the AI explainer.
  3. Apply Fixes: Draft automated GitHub patch pull requests or dispatch incident alerts to team channels.

Interactive Demo

Don't have a log file ready? Head to the Upload page and select one of the three preloaded scenario logs (E-commerce Outage, Intrusion Audit, or Kubernetes Pod issues) to instantly explore the dashboard capabilities.

Supported Log Formats

LogiQ utilizes a lexical parse engine to structure lines. Standard patterns are extracted automatically, falling back to line-by-line checks on unknown patterns.

Supported Schemas

  • Syslog & Kernel Logs: Standard RFC 5424 time formats with host/service tags.
  • Nginx & Apache Access Logs: Tracks resource paths, HTTP status (e.g. 403, 429, 500), and latencies.
  • JSON-structured Logs: Structured fields like `{"timestamp": ..., "level": ..., "message": ...}`.
  • Generic Timestamp Text: Formats matching standard ISO, UTC, or localized datetime strings.

Sample Format Structure

2026-06-26T08:02:15.654Z ERROR [db-pool] Connection refused: db-primary:5432
2026-06-26T08:03:00.000Z CRITICAL [memory] Out of memory: Java heap space

Self-Healing Integrations

Configure external integrations within the application to automatically notify your dev team or draft code patches for detected anomalies.

Slack & Discord Notifications

Enter webhook URLs on your Profile page settings tab. Once saved, you can click "Alert" inside the Anomaly details panel to instant-broadcast critical system incidents with description details and stack messages.

GitHub Pull Requests

Using the GitHub integration, LogiQ connects directly via repositories to draft patch files. For example, if a log indicates a configuration permission issue or database address error, the AI engine drafts a code change and pushes a Pull Request directly to your repository branch.

API Reference

Drive log analysis programmatically using LogiQ API endpoints. Authenticate by passing your API Key token in requests headers.

1. Upload and Analyze Log

POST /api/upload

Send a multipart form-data payload containing the log file.

curl -X POST -H "Authorization: Bearer " \
     -F "file=@production.log" \
     http://localhost:5000/api/upload

2. Trigger AI Explanation

POST /api/explain

Query explanation database cache or LLM engine for specific anomaly. JSON body:

curl -X POST -H "Content-Type: application/json" \
     -d '{"anomaly_id": 42}' \
     http://localhost:5000/api/explain
